## 2025-04-04 - Optimize styled-components dynamic props
**Learning:** In styled-components, passing rapidly changing or highly variable dynamic props (like specific background images or colors for individual cards) directly to the template literal generates a new CSS class for *every* unique value. This bloats the injected `<style>` tags and forces the browser to recalculate styles excessively, degrading performance.
**Action:** Use `.attrs(props => ({ style: { ... } }))` to inject highly dynamic props as inline styles or CSS variables. This ensures the styled-component generates a single, static CSS class that is reused across all instances, significantly reducing overhead and improving rendering speed. Use CSS variables instead of inline styles if the dynamic value needs to be used in pseudo-elements (`::before`) or overridden by media queries, because inline styles have higher specificity and cannot directly target pseudo-elements.
